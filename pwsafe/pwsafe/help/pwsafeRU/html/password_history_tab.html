Вкладка «История паролей»



При изменении пароля элемента Password Safe может сохранять предыдущее значение. Это может пригодится в том случае, когда смена пароля на соответствующем сайте завершается ошибкой. Данная настройка указывает, нужно ли по умолчанию включать сохранение предыдущих паролей для новых элементов. Данную настройку можно переопределить при добавлении или редактировании элемента.

Хранить N предыдущих паролей для элемента

Установка данного флажка задаёт количество паролей, сохраняемых для новых элементов по умолчанию.

Управление историей паролей для текущих элементов

Данный раздел позволяет управлять настройками ведения истории паролей для уже созданных элементов:
  • Без изменений — настройки сохранения предыдущих паролей не изменяются.
  • Перестать сохранять предыдущие пароли — равносильно снятию флажка «Хранить N предыдущих паролей» во всех элементах контейнера. Обратите внимание, что существующая история будет потеряна, и изменения паролей в этих элементах не будут отслеживаться до момента повторного включения.
  • Начать сохранять предыдущие пароли — равносильно установке флажка «Хранить N предыдущих паролей» во всех элементах контейнера. Элементы, история для которых не сохранялась, будут запоминать указанное количество предыдущих паролей. Элементы, ведущие историю паролей, продолжат сохранять изменения, при этом количество сохраняемых паролей не будет изменено.
  • Установить максимальное количество сохраняемых паролей равным указанному значению — все элементы будут хранить указанное количество паролей. Но, если у каких-либо элементов ранее было указано большее значение, история не будет потеряна.
