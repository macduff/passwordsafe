Параметры командной строки

При запуске Password Safe без параметров выдается запрос пароля для последнего открытого контейнера, или для нового контейнера, если Password Safe не использовался ранее на данном компьютере. Это поведение можно изменить, вызвав Password Safe следующим образом:

  • pwsafe контейнер
    Будет открыт указанный в параметре файл вместо того, что использовался в прошлый раз. Если указано только имя файла (без пути), то поиск файла производится в каталоге из которого было запущено приложение. Обратите внимание, что имя файла нужно заключать в двойные кавычки ("имя файла"), если оно содержит пробелы.
  • pwsafe -r [контейнера]
    Указанный контейнер будет открыт в режиме "только для чтения". Если контейнер не указан, будет запрошено имя контейнера, который затем будет открыт в режиме "только для чтения".
  • pwsafe -v контейнера
    После ввода пароля для контейнера будет проведена проверка на наличие несоответствий. Данный режим полезен в случае, если контейнер был создан в стороннем приложении.
  • pwsafe -e имя_файла
    Указанный файл будет зашифрован с использованием ключа, полученного из введенного пароля. Примечание: Файл может быть любым. Имя зашифрованного файла останется тем же, а в качестве расширения будет добавлено ".PSF".
  • pwsafe -d имя_файла
    Указанный файл будет расшифрован с использованием ключа, полученного из введенного пароля. Внимание: Данный метод применим только к файлам ранее зашифрованным с использованием параметра "-e".
  • pwsafe -c
    В данном случае приложение будет запущено без контейнера, в том числе не будет выведен стандартный диалог (для открытия контейнера можно использовать меню "Файл").
  • pwsafe -s [контейнер]
    Приложение будет запущено "молча" - в свернутом виде и без контейнера (если он не указан в параметрах). При развертывании окна будет выведен диалог открытия контейнера (параметр предназначен для запуска Password Safe при входе в систему через ярлык автозапуска). Внимание: в данном случае значок приложения будет помещен в область уведомлений (трей).
  • pwsafe -m
    Параметр аналогичен '-c', но, дополнительно, запуск будет произведен в свернутом виде.
Также допустимы следующие параметры:
  • -u имя_пользователя
    При запуске приложение будет использовать настройки, заданные для пользователя имя_пользователя, вместо настроек пользователя, вошедшего в систему.
  • -h имя_хоста
    При запуске Password Safe будет использовать настройки, указанные для имя_хоста, вместо настроек для текущей машины.
  • -g файл_конфигурации
    В данном случае указанный файл будет использован для загрузки и сохранения настроек (вместо файла по умолчанию "pwsafe.cfg"). Если указано только имя файла (без пути), то поиск файла производится в каталоге из которого было запущено приложение. Обратите внимание, что имя файла нужно заключать в двойные кавычки ("имя файла"), если оно содержит пробелы.
Данные параметры могут оказаться полезной если вы желаете использовать одни и те же настройки на разных машинах, например, при запуске Password Safe со съемного диска.
