Резервное копирование и восстановление контейнеров

Так как информация, хранящаяся в контейнере с паролями обычно является важной, Password Safe предоставляет несколько способов резервного копирования и восстановления контейнеров. Данный раздел описывает методы ручного создания резервных копий. Также можно настроить Password Safe таким образом, чтобы резервные копии создавались автоматически при сохранении контейнера. Такие резервные копии называются промежуточными и описываются здесь.

Создание резервной копии

Создание резервной копии текущего контейнера:

  1. Нажмите Создать резервную копию
  2. Выберите расположение файла резервной копии
  3. Нажмите Сохранить

Примечание: настоятельно рекомендуется делать резервные копии паролей как можно чаще. Этот поможет избежать потери списка паролей в случае системного сбоя или повреждения файлов. Резервная копия использует для шифрования тот же мастер-пароль, что и текущий контейнер.

Восстановление из резервной копии

Восстановление контейнера из резервного копии:

  1. Нажмите Восстановление из резервной копии
  2. Перейдите в папку с резервной копией и выберите файл для восстановления
  3. Нажмите Открыть
    Будет запрошен пароль для контейнера.
  4. Введите пароль и нажмите OK
Примечания:
  • Чтобы восстановленный контейнер сохранился со стандартным для Password Safe расширением (.psafe3), выберите Сохранить или Сохранить как в меню Файл. Перед выходом из приложения будет предложено сохранить контейнер, если вы не сделали этого ранее.
  • Кроме того Password Safe автоматически создает копию контейнера перед сохранением изменений. Копия создается в том же каталоге, что и текущий контейнер, имеет то же имя и отличается лишь расширением (".ibak" вместо ".psafe3").
