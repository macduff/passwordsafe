Automatische Eingabe

Die Automatische Eingabe stellt eine einfache und schnelle Methode zur Verfügung um Benutzernamen und Passwörter einzugeben. Mit einem einzigen Klick werden Benutzername und Passwort angegeben und die Daten werden zur Freigabe dem System übergeben. Standardmäßig ist die Formel für den automatisierten Tastenanschlag wie folgt: Benutzername, gefolgt vom TAB Zeichen, gefolgt vom Passwort, noch ein TAB Zeichen und dann die Eingabe-Taste (Benutzername TAB Passwort TAB ENTER). Dies funktioniert für die meisten WEB Seiten die eine Authentifikation mit Benutzername/Passwort benötigen.

Für WEB Seiten oder Anwendungen die andere Daten erfordern, ist es möglich das Verhalten der 'Automatischen Eingabe' anzupassen wie beschrieben in Automatische Eingabe anpassen weiter unten. Bemerkung: man kann auch den Text für die standardmäßige Automatische Eingabe im Menü Verwalten->Optionen->Verschiedenes anpassen. Dies kann z.B. dann nützlich sein wenn die Automatische Eingabe für Ihre Maschine zu schnell ist.

Automatische Eingabe anwenden

  1. Setze den Kursor im Feld Benutzername des Formblattes
  2. Maximiere oder öffne Password Safe
  3. Wähle den Passworteintrag aus
  4. Ctrl -T oder Rechts-Klick und wähle Autom. Eingabe ausführen aus oder Klick in der Toolleiste den entsprechenden Knopf zum Beginnen.

Automatische Eingabe anpassen

Für WEB Seiten oder Anwendungen bei denen das Feld 'Benutzername' nicht vom Feld 'Passwort' und vom Feld 'Anmelden' gefolgt wird, können Sie das Verhalten abändern indem sie im Feld "Automatische Eingabe" Text ausfüllen der das erhoffte Verhalten spezifiziert. Dies geht wie folgt:

  • \u wird ersetzt mit dem Benutzername vom Eintrag.
  • \p wird ersetzt mit dem Passwort vom Eintrag.
  • \g wird ersetzt mit der Gruppe verknüpft mit dem Eintrag.
  • \i wird ersetzt mit der Titel verknüpft mit dem Eintrag.
  • \o wird ersetzt mit dem Inhalt des Feldes Notizen (möglicherweise mehrzeilig).
    • Alle Zeilenende Zeichen "\n", "\r" oder "\r\n" werden umgeformt zu "\r" (carriage return).
    • Es erfolgt keine Ersetzung für Escape Sequenzen im Feld Notizen (z.B. "\u", "\p" usw.) außer für die "\t" Sequenz, die ersetzt wird mit dem TAB Zeichen.
  • \oNNN wird ersetzt mit dem Inhalt von der nten Zeile im Feld Notizen.
    • NNN kann aus 1 bis 3 Ziffern bestehen (z.g., \o5 für die 5te Zeile, \o45 für die 45ste, usw.
    • Zeilenende Zeichen sind nicht eingeschlossen.
    • wenn NNN größer ist als die Anzahl Zeilen im Feld Notizen, dann hat dies keine Auswirkung.
    Hinweis: dies ist eine bequeme Möglichkeit Autom. Eingabe zu unterstützen für Anwendungen die ein zweites Passwort verlangen oder andere beliebige Text als Teil des Authorisierungsprozesses benötigen.
  • \b wird ersetzt mit der Backspace-Taste.
  • \t wird ersetzt mit der TAB Taste, um das Feld zu überspringen.
  • \n wird ersetzt mit der ENTER Taste, um die Taste zu 'Drücken'.
  • \\ wird ersetzt durch die Gegenschrägstrich (\) Taste.
  • \dNNN setzt die Verzögerung zwischen Zeichen auf NNN Millisekunden, anstelle von 10 (Standard).
    Anmerkung: man kann mehrere \d Kommandos ketten. Z.B. \d999\d999\d999\d010 ergibt 3 Sekunden Verzögerung (3.007, um exakt zu sein). Das \d Kommando setzt sowohl die Verzögerung zwischen den Zeichen als auch die anfängliche Verzögerung.
  • Sämtliche andere Text wird so wie er ist eingegeben.

Beispiel: Folgendes kann verwendet werden wo das Bankkonto (12345) vor dem Benutzernamen und das Passwort kommen muss:

12345\t\u\t\p\t\n

Anmerkung: Standardmäßig wird das Feld 'Automatische Eingabe' in den Menüpunkten Einfügen und Bearbeiten nicht angezeigt. Um auf das Feld 'Automatische Eingabe' zuzugreifen sollten sie das Feld "Mehr>>" anklicken im jeweiligen Dialog Feld.

Wenn sie Problem mit der 'Automatischen Eingabe' haben, wäre es empfehlenswert Popup-Blocker von dritten Hersteller abzuschalten. Einige Benutzer haben über Probleme berichtet wegen der gegenseitigen Beeinflussung von Popup-Blocker und der Funktion 'Automatischen Eingabe'.
